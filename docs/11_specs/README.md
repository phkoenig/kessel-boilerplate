# 11_specs — Feature-Specs

Output des Skills `feature-specing` (`.claude/skills/feature-specing/`).

## Zweck

Implementierungsreife Feature-Specs mit Scope, User Stories, Akzeptanzkriterien und Risiken.

## Naming

Living Docs ohne Datum: `<slug>.md` (kebab-case, max 60 Zeichen).
Status im Frontmatter oder am Dokumentenanfang: `Draft | Review | Approved`.

## Archiv

Verworfene oder durch Nachfolger ersetzte Specs in `archive/` verschieben.
